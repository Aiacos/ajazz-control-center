# FAQ

## Is this an official AJAZZ product?

No. AJAZZ Control Center is an independent open-source project, not
affiliated with, endorsed by or supported by AJAZZ. All trademarks belong
to their respective owners.

## What license is the code under?

GPL-3.0-or-later. Any derivative work that is distributed must publish
its full source under the same license. See
[LICENSE](https://github.com/Aiacos/ajazz-control-center/blob/main/LICENSE).

## Why GPL and not MIT?

Reverse-engineered device protocols are a commons. Copyleft prevents a
vendor from taking the work, closing it and using it against the
community that produced it. Plugins are loaded as separate Python
packages and are not considered derivative works.

## Can I use this commercially?

Yes, as long as you comply with the GPL. Streaming with it, using it in
a studio, or bundling it on your own machines is all fine. Selling a
proprietary fork is not.

## Will there be a mobile companion app?

Not in the short term. The focus until v1.0 is desktop feature parity
with the vendor tool.

## Why Qt 6 and not Electron / Tauri / Flutter?

- Native performance for per-key rendering and HID I/O.
- Single binary, no browser runtime.
- First-class Linux packaging (Flatpak, RPM, DEB) is painful with web
  stacks.
- The maintainer has ~10 years of Qt experience.

## Why embed Python rather than expose Lua / JavaScript / WebAssembly?

The plugin ecosystem is the main differentiator. Python has the widest
library coverage for the kinds of things stream-deck users automate
(OBS, home assistant, Spotify, Git, Figma, etc.) and the best tooling
for non-professional developers.

## Does it work over Bluetooth / 2.4 GHz dongles?

Wired USB first, because capture and debugging are tractable. Wireless
variants are on the roadmap once the wired protocol is stable.

## Does it read my keystrokes?

Only the ones generated by, or destined for, an AJAZZ device. The app
does not install a system-wide keylogger. Event capture is scoped to
HID reports originating from connected AJAZZ devices.

## How are updates delivered?

From the [Releases page](https://github.com/Aiacos/ajazz-control-center/releases):
native packages on Linux (dnf/apt/flatpak auto-update), MSI on Windows
(update channel on roadmap), DMG on macOS (Sparkle integration on
roadmap).

## Can it flash firmware?

No. Firmware flashing is considered out of scope — the risk of bricking
a device from reverse-engineered flashing code is too high. Use the
vendor tool for firmware updates.

## Will it coexist with the vendor app?

They can both be installed, but only one should talk to the device at a
time. On Linux and macOS this is enforced by the HID claim; on Windows
the vendor app will usually release the handle when minimised.

## How do I report a bug?

Open a
[Bug Report](https://github.com/Aiacos/ajazz-control-center/issues/new?template=bug_report.yml)
with the output of `acc --log-level=debug --dump-capabilities` attached.

## How do I request a device?

Open a
[Device Request](https://github.com/Aiacos/ajazz-control-center/issues/new?template=device_request.yml)
with a USB capture from the vendor app.
