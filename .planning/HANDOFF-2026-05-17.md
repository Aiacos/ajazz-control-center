## Anti-features — DELIBERATELY skip (preserve in roadmap §4)

Confirmed worth NOT copying after this session's RE pass. Future contributors:
1. Cloud login (api.rongyuan.tech, qmk.top, hotspot Alibaba CDN).
2. Telemetry (universal-analytics, machine-ID generator, watchSystemInfo gRPC stream).
3. `getWeather` gRPC → leaks user-typed address to vendor server.
4. Recoil-control / rapid-fire opcodes — anti-cheat liability.
5. Plaintext gRPC localhost listener with no auth (security regression vs our in-process `ITransport`).
6. Custom encryption layer over SQLite (over-engineered threat model).
7. Global low-level keyboard hook (Win32 `SetWindowsHookEx`) — only acceptable as opt-in user-toggleable.
8. Alibaba Cloud OSS plugin store + auto-update CDN.
9. 2-process Rust+Electron split (vendor AJ159 architecture).
10. Custom UI framework like `mui.dll` (AK980 PRO) — Qt6/QML is the right answer.
11. Dynamic `LoadLibraryA("hid.dll") + GetProcAddress` indirection (vendor AK980 PRO) — link hidapi normally.
12. HTTP-spawn external `FirmwareUpdateTool.exe` — do in-app DFU with progress and rollback path.
13. 200 ms polling loop for foreground-app detection — use `QSocketNotifier` / Win32 completion ports / equivalent.
14. Vendor's `commit()` no-op on mouse — vendor has no separate commit step; opcode 0x54 DPI writes are immediately persistent.
15. Embedded sled / SQLite for tiny config payloads — JSON profiles + Qt6 `QSettings` is the right shape.
