# Quick Task 260513-v6b: Plugin Store Phase B

**Parent:** Resolves [260513-u0b HIGH-1 + HIGH-2 + LOW-2](../260513-u0b-streamdock-plugins-diagnostic/260513-u0b-FINDINGS.md) — the layout-related findings the parent diagnostic explicitly deferred to Phase B.

## Goal

Replace QML-side filtering (`tile.matches` + `visible:false; width:0; height:0` + manual `visibleCount`) with a C++ `QSortFilterProxyModel` so the GridView only sees matching rows and the layout reflows correctly.

## Why

The diagnostic identified two interacting issues in the current QML approach:

1. **HIGH-1** — `grid.visibleCount` is maintained by per-delegate `Component.onCompleted` / `onDestruction` increment-decrement on a recyclable GridView. Recycled delegates drift the counter, breaking `EmptyState.visible: visibleCount === 0`.
1. **HIGH-2** — `visible: false; width: 0; height: 0` leaves cell-slots in the GridView (cellWidth/cellHeight are fixed); visible tiles get sparse "phantom gaps."

Both vanish if the GridView's `model` is a filtered proxy. Single source of truth for matching rows: `proxy.rowCount()`.

## Scope

- **In:** New `PluginCatalogProxyModel : QSortFilterProxyModel` C++ class declared as `QML_NAMED_ELEMENT(PluginCatalogProxy)`. PluginStore.qml refactor to use the proxy. Unit tests covering tab + query filtering.
- **Out:** `activeTab` int → string migration (MEDIUM-1 from diagnostic — separate concern). Performance-tuning at scale beyond making the GridView correct. Source-side changes to `PluginCatalogModel`.

## Tasks

### Task 1 · Create `PluginCatalogProxyModel`

**Files (new):**

- `src/app/src/plugin_catalog_proxy_model.hpp`
- `src/app/src/plugin_catalog_proxy_model.cpp`

**Design:**

```cpp
class PluginCatalogProxyModel : public QSortFilterProxyModel {
    Q_OBJECT
    QML_NAMED_ELEMENT(PluginCatalogProxy)
    Q_PROPERTY(int activeTab READ activeTab WRITE setActiveTab NOTIFY activeTabChanged)
    Q_PROPERTY(QString query READ query WRITE setQuery NOTIFY queryChanged)
    Q_PROPERTY(int count READ count NOTIFY countChanged)

public:
    enum Tab {
        AllTab = 0,
        InstalledTab = 1,
        StreamdockTab = 2,
        OpenDeckTab = 3,
        CommunityTab = 4,
    };
    Q_ENUM(Tab)

    explicit PluginCatalogProxyModel(QObject* parent = nullptr);
    int activeTab() const noexcept;
    void setActiveTab(int tab);
    QString query() const noexcept;
    void setQuery(QString const& q);
    int count() const;

signals:
    void activeTabChanged();
    void queryChanged();
    void countChanged();

protected:
    bool filterAcceptsRow(int sourceRow, QModelIndex const& sourceParent) const override;

private:
    int m_activeTab = AllTab;
    QString m_query; // already-lowercased
};
```

**filterAcceptsRow logic** (mirrors `PluginStore.qml`'s `rowMatches`):

1. Tab filter:
   - `AllTab` → accept
   - `InstalledTab` → require `InstalledRole == true`
   - `StreamdockTab` → require `SourceRole == "streamdock"`
   - `OpenDeckTab` → require `SourceRole == "opendeck"`
   - `CommunityTab` → require `SourceRole == "community"`
1. If query empty → accept (after tab passed).
1. Query filter: case-insensitive substring across `NameRole`, `DescriptionRole`, any string in `TagsRole`.

**Setters:** `setActiveTab` / `setQuery` call `invalidateFilter()`, emit `*Changed`, and `countChanged()`. `setQuery` lowercases internally.

**countChanged wiring:** Connect in constructor to `rowsInserted`, `rowsRemoved`, `modelReset` (which fire as side-effects of invalidateFilter when matches change), so `count` Q_PROPERTY stays current.

**Done when:** Header + source compile clean, class instantiable from QML, role-name access pattern works.

   - `readonly property bool matches: root.rowMatches(...)` (~3 lines)

1. Update `EmptyState.visible: grid.visibleCount === 0` → `catalogProxy.count === 0`.
1. Delete the QML `rowMatches()` function (now in C++).

**Done when:** PluginStore.qml is shorter (~30 LoC net delete) and uses the proxy. `qmllint` clean.
