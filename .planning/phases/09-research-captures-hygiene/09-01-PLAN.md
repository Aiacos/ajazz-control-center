Per D-02 (per-extension blocklist + 10 KB size guardrail under `.planning/research/captures/`):

1. Write `scripts/reject-raw-captures.sh`:

   - `#!/usr/bin/env bash` shebang. `set -euo pipefail`.
   - Reads staged file paths from `$@` (pre-commit passes them as positional args
     when `pass_filenames: true`).
   - For each path:
     - If basename matches `*.pcap` or `*.pcapng` (case-insensitive — use
       `shopt -s nocasematch` or `[[ "${p,,}" == *.pcap || "${p,,}" == *.pcapng ]]`),
       emit a structured rejection message to stderr and set `bad=1`.
     - If path starts with `.planning/research/captures/` AND is not exactly
       `.planning/research/captures/.gitignore` or `.planning/research/captures/README.md`
       AND is binary AND size >10240 bytes, emit a rejection message and set `bad=1`.
       Use `file --mime-encoding "$p" 2>/dev/null | grep -q binary` for the binary
       check; use `wc -c < "$p"` for size. Guard both with `[[ -f "$p" ]]`.
   - Exit `1` if `bad=1`, else exit `0`.
   - The rejection message MUST cite:
     - The offending path.
     - The reason (extension blocklist OR captures-sink size guardrail).
     - The remediation: "See docs/policies/capture-data-hygiene.md and use
       scripts/hex-to-cpparray.py to produce a sanitised fixture under
       tests/integration/fixtures/<codename>/."
   - ASCII-only output (CLAUDE.md hard rule — Win32 CMD codepage mangling on
     Windows CI shells that ever invoke the hook locally).
   - `chmod +x` after writing (pre-commit's `check-executables-have-shebangs`
     enforces consistency; the easiest path is execute permission + shebang).

1. Edit `.pre-commit-config.yaml`:

   - Locate the existing `repo: local` block that hosts `regenerate-docs` and
     `validate-cmake-presets` (around the bottom of the file, just above the
     `conventional-pre-commit` block).
   - Append a new local hook entry to the SAME `repo: local` block (do not
     create a second `repo: local` entry — cmake-format / yamllint allow it but
     it's noise):
     ```yaml
       - id: reject-raw-captures
         name: Reject raw USB capture files (CAPTURE-01 / Pitfall 17)
         entry: scripts/reject-raw-captures.sh
         language: system
         pass_filenames: true
         always_run: true
         # Run on every commit attempt — we want to catch additions of pcap
         # files regardless of extension prefix, including under directories
         # the global `exclude:` block (legacy `docs/protocols/captures/.*`)
         # would otherwise suppress.
     ```
   - DO NOT remove the existing top-level `exclude:` block that lists
     `docs/protocols/captures/.*` and `.*\.pcapng` — those are lint exclusions,
     not commit rejections, and removing them would generate noise on the legacy
     captures path. The new hook is the authoritative rejection layer.

1. After writing both files, run `pre-commit run --all-files reject-raw-captures`
   manually to confirm it exits 0 on the clean tree.

DO NOT add a smoke-test pcap file to the repo to test the rejection — that would
defeat the purpose. The hook's correctness is verified in Task 3.
</action>
