\<usbrply_json_shape>
usbrply `-j` output is a JSON document whose payload-bearing entries look like:

```json
{
  "packs": [
    {"type": "controlWrite", "data": "020b000000000000444c494d6718e865", "request_type": 0x21, ...},
    {"type": "interruptIn",  "data": "00000000000000000007000000000000", ...},
    {"type": "interruptOut", "data": "....", ...}
  ]
}
```

The `data` field is hex bytes (no `0x`, no spaces, all lowercase) of the HID
payload, NOT the URB header. The script extracts the `data` strings for every
packet of a chosen type (default: all data-bearing packets) and emits ONE
`std::array` per script invocation containing the concatenation OR a single
selected packet depending on `--mode` (single by default — usbrply runs are
typically narrowed to one event before conversion via the tshark filter in
CAPTURING.md §5).
\</usbrply_json_shape>

1. Write `scripts/hex-to-cpparray.py` with the following structure:

   - Shebang: `#!/usr/bin/env python3`.
   - SPDX header: `# SPDX-License-Identifier: GPL-3.0-or-later`.
   - Module docstring summarising purpose, contract, anti-features
     (no libpcap dep; uses only stdlib).
   - Imports: stdlib only — `argparse`, `datetime`, `json`, `pathlib`,
     `re`, `sys`. NO third-party imports.
   - Constants:
     - `MAX_BYTES_PER_LINE = 8` (formatting — emit 8 hex bytes per source line).
     - `HEX_RE = re.compile(r"^[0-9a-fA-F]+$")` (validation).
   - Functions:
     - `parse_args(argv: list[str]) -> argparse.Namespace`:
       - `input` (positional, path to usbrply JSON, "-" for stdin).
       - `--device` (required, codename; ASCII-only,
         `[a-z0-9_]+` regex enforcement — Pitfall 32 ASCII-only codenames).
       - `--capture` (required, human-readable label; ASCII; will be
         normalised to snake_case for the identifier).
       - `--packet-index` (optional int, default 0; which data-bearing
         entry from the JSON to convert).
       - `--type` (optional, default `any`; filter to one of
         `controlWrite`/`controlRead`/`interruptIn`/`interruptOut`/`any`).
     - `normalise_label(s: str) -> str` — converts the `--capture` value
       to UPPER_SNAKE_CASE (e.g. `image-upload-first-chunk` → `IMAGE_UPLOAD_FIRST_CHUNK`).
     - `normalise_device(s: str) -> str` — same shape, UPPER_SNAKE_CASE.
     - `select_packet(usbrply_doc: dict, index: int, type_filter: str) -> bytes`:
       - Iterates `usbrply_doc["packs"]`, filters by `type` if not `any`,
         picks the `index`-th match, returns `bytes.fromhex(packet["data"])`.
       - Raises `ValueError` with a clear message if `data` field is missing,
         empty, or fails `HEX_RE`.
       - Raises `ValueError` if no packet matches the filter or `index`
         is out of range.
     - `emit_header(name: str, payload: bytes, device: str, label: str) -> str`:
       - Returns the full header content as a string.
       - Header layout matches the `<interfaces>` target output exactly:
         SPDX line, AUTOGENERATED notice, source metadata, generation
         timestamp (UTC ISO 8601 to second precision — DO NOT include
         microseconds; reproducible-ish), `#pragma once`, `#include`
         lines, namespace open, the `inline constexpr std::array<std::uint8_t, N>`
         declaration with 8 bytes per source line, namespace close.
       - Identifier shape: `<DEVICE>_<LABEL>_BYTES` (e.g.
         `AKP03_VARIANT_3004_IMAGE_UPLOAD_FIRST_CHUNK_BYTES`).
     - `main(argv: list[str]) -> int`:
       - Parses args, opens input (`-` → stdin), loads JSON, calls
         `select_packet`, calls `emit_header`, prints to stdout, returns 0.
       - Catches `ValueError` / `json.JSONDecodeError` / `FileNotFoundError`
         and prints a clear stderr message + returns non-zero.
   - `if __name__ == "__main__": sys.exit(main(sys.argv[1:]))`.

1. The output header MUST:

   - Have `#pragma once` (not include guards).
   - Use `inline constexpr` (NOT `static constexpr` — see precedent at

1. The script must pass `ruff check --fix scripts/hex-to-cpparray.py` and

## Trust Boundaries

| Boundary                         | Description                                                                                     |
| -------------------------------- | ----------------------------------------------------------------------------------------------- |
| user-supplied JSON → script      | Untrusted file path + content (could be malformed, oversized, or hostile)                       |
| script output → committed header | Hex bytes flow into a `.h` file that ends up in the test binary — any malformed bytes propagate |

## STRIDE Threat Register

| Threat ID        | Category               | Component                                                                                    | Disposition | Mitigation Plan                                                                                                                                                                |
| ---------------- | ---------------------- | -------------------------------------------------------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| T-09-11          | Tampering              | Hostile usbrply JSON with non-hex characters in `data` field could emit invalid C++          | mitigate    | `HEX_RE` validates every `data` string; `ValueError` raised on first non-match before any output is written                                                                    |
| T-09-12          | Information Disclosure | Output header could leak operator path / metadata via `--device` or `--capture` content      | accept      | both values are operator-controlled inputs; ASCII-only regex on `--device` (codename) prevents injection; `--capture` is free-form but only appears inside a `//` comment line |
