   `#ifdef AJAZZ_REAL_HARDWARE` guard so that without the flag the TU compiles to a single
   no-op/SKIP and CANNOT open a real device. Inside the guard, define one TEST_CASE with the hidden
   Catch2 tag set `"[.][hardware][akp03][display]"` (ASCII-only per CLAUDE.md — no em-dash/arrow;

1. Test body (operator-run, real `0300:3004` attached):
   - Construct the DeviceDescriptor for `akp03_variant_3004` and a DeviceId with vid 0x0300 /
     pid 0x3004 (resolve serial from enumeration if needed). Call `makeAkp03(...)` and `open()`.
     If no device is present, `SKIP("no 0300:3004 attached")` (Catch2 SKIP) — do NOT fail.
   - Run 100 iterations of: build a small synthetic RGBA8 image (vary a pixel each iteration so the
     payload is non-constant), `setKeyImage` it to a rotating key index 0..5, `flush()`, then
     assert `isOpen()` still true and a follow-up cheap command (`setBrightness` round-trip or a
     `clearKey`) still returns without timing out. The "power-cycle" dimension is operator-driven:
     the runbook (Task 3) instructs the operator to physically power-cycle the device every N
     iterations; the test detects a hang by a write/read timeout or `isOpen()==false` after a
     cycle and FAILS with a clear message ("device hung after image burst N — Pitfall 18").
   - Use a generous per-call timeout; a hang manifests as a transport exception or a stuck write.
     Catch transport exceptions and FAIL with the iteration count so the operator can correlate.
   - `close()` cleanly at the end.
1. ASCII-only test name + tags (CLAUDE.md: ctest filter passes through the Win32 CMD codepage).
   No nlohmann. The test owns no fixtures under `.planning/research/captures/` (CAPTURE-01 hygiene).
