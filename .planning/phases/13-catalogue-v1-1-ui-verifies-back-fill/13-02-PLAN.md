VERIFY-01 (Sync-button visibility tracks hasClock):

- Steps: launch app; observe the device sidebar rows.
- Expected: the "Sync time to device" ToolButton (refresh-arrow icon; tooltip
  "Sync time to device") is VISIBLE on rows whose device advertises hasClock=true and
  ABSENT on rows whose device has hasClock=false.
- Acceptance: PASS iff the button is hidden on akp03_variant_3004 (post-DEVICES-05
  demotion) AND hidden on ak980pro (post-DEVICES-06 demotion) AND visible on at least
  one v1.1 catalogue device that still advertises hasClock (e.g. an AKP153-family
  streamdeck whose capabilities still include clock). If a demotion has not yet landed,
  record BLOCKED + the dependency, do not FAIL.

VERIFY-02 (auto-sync toggle survives restart + 300ms arrival firing):

- Steps: Settings -> Time sync -> toggle "Auto-sync time on device connect" ON; fully
  quit and relaunch the app; re-open Settings.
- Expected (persistence): the switch is still ON after restart (backed by QSettings key
  Time/AutoSync).
- Steps (firing path): with auto-sync ON and a capable device, unplug+replug that device
  (or trigger an arrival); inspect the app log.
- Expected (firing): a log line shows the 300ms-debounced arrival push attempt for the
  arriving codename (capability re-validated at firing time per v1.1 D-02).
- Acceptance: PASS iff the toggle state survives restart AND the arrival path fires
  ~300ms after arrival in the log.

VERIFY-03 (NotImplemented -> exclamation glyph + tooltip, never a false success toast):

- Pre-condition: a device row whose backend returns NotImplemented from
  IClockCapable::setTime (per ARCH-05 default verdict, akp03_variant_3004 / ak980pro
  setTime stays NotImplemented unless their captures flipped it).
- Steps: click the row's "Sync time to device" button (if visible) OR observe the row
  after an auto-sync tick on a NotImplemented device.
- Expected: an exclamation "!" glyph appears at the row's trailing edge; hovering it
  shows the tooltip "Time sync not yet supported on this device". For an AUTO-sync tick
  the glyph is the ONLY surface (D-02: no toast). For a MANUAL click a failure toast may
  appear, but it MUST be a failure/NotImplemented message - NEVER a "Time synced"/success
  toast (Pitfall 19 honesty contract).
- Acceptance: PASS iff the exclamation glyph + correct tooltip render AND no "Time
  synced"/success toast ever appears for a NotImplemented result. Any green success
  surface on NotImplemented is an immediate FAIL.

VERIFY-04 (MaturityRole tooltip; all 5 tiers render legibly):

- Steps: hover (long-hover ~800ms) each sidebar row.
- Expected: a tooltip "Maturity: <tier>" appears, where <tier> is the device's

- Acceptance: PASS iff every hovered row shows "Maturity: <correct tier>" and all 5 tier
