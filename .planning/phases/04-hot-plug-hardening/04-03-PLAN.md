<objective>
Add a `HotplugDebouncer` helper class owned by `Application` that coalesces raw `HotplugEvent`s by `(vid, pid, serial)` with a 300ms trailing-edge debounce per D-05 / HOTPLUG-05. Route `Application::onHotplug` through it before any consumer (DeviceModel, future TimeSyncService) sees the event.

Purpose: closes Pitfall 3 (toast-flood / event-storm UX) at the application boundary. Composite-USB Stream Dock arrivals (2 interface `add` events per physical connect on Linux), USB-hub shuffles (4-8 events in \<500ms), and flaky-cable bounces collapse to one downstream event per device per stable transition. Pre-condition for D-03 / Plan 04's fine-grained `dataChanged` propagation â€” without coalescing, even fine-grained updates would thrash.

Output: `HotplugDebouncer` class (~60 LoC), wired into `Application::onHotplug`. Existing per-event behaviour preserved for distinct keys; redundant events on the same key collapse.
</objective>
