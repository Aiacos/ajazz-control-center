    Edit `.github/workflows/ci.yml`. Add a new step in the Linux job (only needs to run on Linux — pure grep on the source tree) AFTER the checkout step but BEFORE the build step (so a violation fails fast without burning a Qt install / compile cycle):

````
```yaml
      - name: Enforce hid_open invariant (Pitfall 11)
        if: runner.os == 'Linux'
        run: |
          # Pitfall 11: hid_open + hid_set_nonblocking must stay paired in
          # src/core/src/hid_transport.cpp. Any new hid_open() call elsewhere
          # in src/ skips the non-blocking flag and breaks I/O-thread shutdown.
          if grep -rn 'hid_open(' src/ | grep -v hid_transport.cpp | grep -v 'hid_open_path('; then
            echo "::error::hid_open() found outside src/core/src/hid_transport.cpp — Pitfall 11 invariant violated."
            exit 1
          fi
          echo "hid_open invariant OK — only hid_transport.cpp opens HID handles."
```

Place the step in the Linux-conditional region of the existing build matrix job (between the existing "Install Linux deps" step and the "Install Qt 6" step is a reasonable home; the grep doesn't need any installed deps). Do NOT add it to the macOS or Windows runs — the Linux job is enough to catch every PR (the source tree is identical across runners).

Verify the rule passes today by running the same grep locally before committing:
```
grep -rn 'hid_open(' src/ | grep -v hid_transport.cpp | grep -v 'hid_open_path('
```
The local run must return zero hits — otherwise we'd be landing the gate red. If it does return hits, document them in the SUMMARY (and either justify or fix them) before merging.
````
