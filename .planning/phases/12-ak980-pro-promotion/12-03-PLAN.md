<objective>
Deliver the KEYBOARD-03 dedicated cmd 0x17 sleep-timer path for `ak980pro`. Today the
sleep-timer rides the settings-batch (`0x07 0x10`, `sleepTimerMinutes` raw minute count) that
shipped under `ISettingsCapable`. KEYBOARD-03 specifies the dedicated cmd `0x17` path with a
DISCRETE picker. This plan adds the `0x17` wire-format builder + envelope + a discrete labeled UI
picker, backed by a MockTransport byte-level test.

Purpose: cmd 0x17 (`FUN_00414020`, ak980pro_vendor.md §3.3) is a single-purpose sleep-timer opcode
distinct from the settings-batch. The user picks an idle-to-OLED-off duration from a fixed list; the
firmware persists it (SAVE 0x02) so it survives power-cycle. This is the honest "discrete picker"
KEYBOARD-03 asks for, replacing the free-form minute value that the batch path carried.

Output: a `CmdSleepTimer` constant + `buildSetSleepTimer` builder + a `setSleepTimer` envelope
method + a discrete QML picker + a MockTransport wire-format test.
