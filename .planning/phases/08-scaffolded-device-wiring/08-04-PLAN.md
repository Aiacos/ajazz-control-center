**Prose emit format (locked):**

- Prefix `works:` items with `✓ `
- Prefix `partial:` items with `⚠ `
- Prefix `pending:` items with `✗ `
- Join items within each sub-list with `, `
- Join the three sections with ` `
- If a sub-list is empty or absent, omit its section entirely.
- Example: `✓ key reads, key images. ⚠ encoder rotation (untested). ✗ firmware update.`
  (Use sentence-style `. ` between the three sections so each reads naturally.)

       ```python
       def _feature_summary_prose(dev: dict) -> str:
           """Format dev['feature_summary'] as ✓/⚠/✗ prose; empty string when absent."""
           fs = dev.get("feature_summary")
           if not fs:
               return ""
           parts = []
           if fs.get("works"):
               parts.append("✓ " + ", ".join(fs["works"]))
           if fs.get("partial"):
               parts.append("⚠ " + ", ".join(fs["partial"]))
           if fs.get("pending"):
               parts.append("✗ " + ", ".join(fs["pending"]))
           return ". ".join(parts) + ("." if parts else "")
       ```
    2. In the per-device row emission code (the lines that currently render `dev.get("notes", "")` into the README/wiki Notes column), compute:
