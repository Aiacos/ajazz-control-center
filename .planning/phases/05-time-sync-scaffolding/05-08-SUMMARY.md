## Decisions Made

- **Integration test scope.** Plan 05-08 spec mentioned `HotplugMonitor::injectEvent` for the integration test, but that shim is AJAZZ_TESTING-gated and the unit test `test_hotplug_harness.cpp` already exercises the HotplugMonitor → debouncer → DeviceModel chain exhaustively (4 SC cases). The integration test instead targets the **new** chain Phase 5 adds: TimeSyncService → DeviceLookup → registry.open() → Akp153Device::setTime() → NotImplemented result. This covers the surface unit tests don't (real factories + real device construction).
- **A-08 enforcement at integration boundary.** D-02 says auto-sync = glyph only, no toast. The toast surface is the `syncFailed` signal. The integration test's TEST_CASE 1 (auto-sync path) asserts `failSpy.count() == 0` after `onDeviceArrived` returns — direct, observable contract that the manual-vs-auto disambiguation is correct at the C++ boundary (the QML latch in Main.qml is the further safeguard at the UI boundary).
- **README + wiki regeneration accepted.** Pre-commit hook regenerated the AUTOGEN blocks in README.md and docs/wiki/Supported-Devices.md from devices.yaml. The diffs are pure additions (new `clock` column in the capability tables); staged and committed in the same Plan 05-08 commit.
