<objective>
Land the QML UI for time-sync — the "Sync time" `ToolButton` on each capable DeviceRow (visible iff the new `deviceHasClock` role is true), the SettingsPage `SwitchDelegate` for "Auto-sync time on device connect", the per-row glyph state, and the toast wiring for manual-sync feedback (D-02: manual = toast + glyph; auto = glyph only).

This plan is a thin pointer-plan: it adopts **Task 6 of the upstream plan at `docs/superpowers/plans/2026-05-13-time-sync.md` (lines 1054-1280)** verbatim, with one observability amendment (toast cap mitigation per Pitfall 3 — see A-05).

**Critical UI contract per D-02:**

- Manual sync (user clicks "Sync time"): success → Toast "Time synced to <model>" + green checkmark glyph (4s auto-dismiss). Failure (NotImplemented) → Toast "Time sync not yet supported on this device" + amber exclamation glyph (6s). Failure (IoError) → Toast "Time sync failed: <detail>" + red exclamation glyph (6s).
- Auto sync (HotplugMonitor → 300ms → TimeSyncService.onDeviceArrived): glyph state updates only. **Never a toast.** Auto-sync IoError does not retry — next Arrived event is the natural retry per design doc.
  </objective>
