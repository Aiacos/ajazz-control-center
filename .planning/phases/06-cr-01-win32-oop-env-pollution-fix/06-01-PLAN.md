```cpp
// src/plugins/src/win32_env_block.hpp
#pragma once
#ifdef _WIN32

#include <map>
#include <string>
#include <vector>
#include <windows.h>

namespace ajazz::plugins {

// RAII helper that builds a UTF-16 environment block suitable for
// CreateProcessW(lpEnvironment=...) with CREATE_UNICODE_ENVIRONMENT.
//
// Lifetime contract: the caller must keep the Win32EnvBlock instance
// alive until CreateProcessW (or CreateProcessAsUserW) returns.
class Win32EnvBlock {
public:
    // Snapshots the parent env at construction time, merges in `overrides`
    // using case-insensitive key matching (override replaces parent value;
    // new override key appends), sorts case-insensitively, and serializes
    // into an internal vector<wchar_t> ending in the mandatory \0\0
    // double-null terminator. Frees the snapshot eagerly.
    explicit Win32EnvBlock(std::map<std::wstring, std::wstring> overrides);

    // Pointer suitable for CreateProcessW(lpEnvironment=...).
    // Buffer is only valid while *this is alive.
    LPVOID data() noexcept;
    SIZE_T size_bytes() const noexcept;

    // Non-copyable, movable.
    Win32EnvBlock(const Win32EnvBlock&) = delete;
    Win32EnvBlock& operator=(const Win32EnvBlock&) = delete;
    Win32EnvBlock(Win32EnvBlock&&) noexcept = default;
    Win32EnvBlock& operator=(Win32EnvBlock&&) noexcept = default;

private:
    std::vector<wchar_t> m_block;  // KEY=VALUE\0KEY=VALUE\0...\0\0
};

}  // namespace ajazz::plugins

#endif  // _WIN32
```

</interfaces>

<pitfalls>
<!-- All mandatory implementation rules from PITFALLS.md Pitfall 5. -->

1. **Block MUST end in `\0\0`** (two consecutive wide nulls). After the last `KEY=VALUE\0` entry, append one more `L'\0'`. Verify in implementation: the final two `wchar_t` of `m_block` are both `0`.
1. **Sort case-insensitively** by key (`_wcsicmp` on the substring up to the first `=`). Windows requires sorted blocks; incorrect sort order = silent env corruption in the child.
1. **Preserve `=`-prefixed entries verbatim** at the FRONT of the block. `GetEnvironmentStringsW` returns drive-letter-current-directory entries like `=C:=C:\foo` — these MUST come first and MUST NOT be touched by override merging.
1. **Free the snapshot via `FreeEnvironmentStringsW`** — NEVER `delete[]`, NEVER `LocalFree`, NEVER leak. Free it inside the constructor once entries are copied into `m_block`.
1. **Lifetime:** the `Win32EnvBlock` instance MUST outlive the `CreateProcessW` call. Declare on the stack of the spawn function before calling spawn.
1. **`bInheritHandles=TRUE` is unchanged.** Pitfall 5 explicitly warns: do NOT modify `bInheritHandles` while landing CR-01. Both call sites already pass `TRUE` (lines 547, 558) — leave them.

```
Constructor algorithm:
1. Call `GetEnvironmentStringsW()` to snapshot parent env. Walk the `\0`-terminated list (each entry ends at `\0`, list ends at `\0\0`).
2. Partition: entries beginning with `L'='` (drive-letter-current-dir, e.g. `=C:=C:\\foo`) go into a separate `frontEntries` vector verbatim — they MUST be preserved at the FRONT of the output block and MUST NOT participate in override merging.
3. For each non-`=`-prefixed parent entry, find the first `L'='` to split into key/value. Look up the key in `overrides` using `_wcsicmp` (case-insensitive). If matched: emit `overrideKey + L"=" + overrideValue` (use the OVERRIDE's key casing, not the parent's), and remove that entry from a working copy of `overrides` so it isn't appended again. If no match: emit the parent entry verbatim.
4. Append any remaining `overrides` entries (new keys not in parent env) as `key + L"=" + value`.
5. Sort the combined non-`=`-prefixed entry list case-insensitively by the substring up to the first `=` (use `std::sort` with a lambda calling `_wcsicmp`).
6. Serialize into `m_block`: first the `frontEntries` (each followed by one `L'\\0'`), then the sorted main entries (each followed by one `L'\\0'`), then a final terminating `L'\\0'` (so the block ends in `\\0\\0`).
7. Call `FreeEnvironmentStringsW(snapshot)` before returning from the constructor.

Accessors: `data()` returns `static_cast<LPVOID>(m_block.data())`; `size_bytes()` returns `m_block.size() * sizeof(wchar_t)`.

Whole file Win32-only via `#ifdef _WIN32` / `#endif` guards. Include `<windows.h>`, `<algorithm>`, `<cwchar>` for `_wcsicmp`. Never include this header from any cross-platform TU. Per D-01, no destructor body needed (the snapshot is freed in the ctor; `m_block` cleans itself).
```
