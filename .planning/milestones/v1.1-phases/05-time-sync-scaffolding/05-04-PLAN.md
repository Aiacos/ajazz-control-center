**Amendment to Task 4 (this plan only):**

- **A-04 (D-01 amendment 3) — shared_ptr cache integration.** The upstream plan (written before Phase 4 D-06) specified raw `IDevice*` lookup. Post-Phase-4, `DeviceRegistry::open()` returns `std::shared_ptr<IDevice>` from a weak_ptr cache. `TimeSyncService::setSystemTimeOn(deviceId)` MUST:

  1. Call `auto ptr = m_registry.open(deviceId);` and **capture the shared_ptr in a local variable** for the lifetime of the function call.
  1. `auto* clk = dynamic_cast<IClockCapable*>(ptr.get());`
  1. Within ≤3 lines: `if (!clk) { emit syncFailed(deviceId, "not_capable", ""); return; }` (Pitfall 2).
  1. `auto result = clk->setTime(std::chrono::system_clock::now());`
  1. Switch on result and emit the appropriate signal.
  1. The `shared_ptr` ptr falls out of scope at function exit; the weak_ptr cache evicts passively per Phase 4 D-06.

  And in `onDeviceArrived(deviceId)`:

  - `QTimer::singleShot(300ms, this, [this, deviceId]{ ... });` — the lambda captures `deviceId` BY VALUE (NOT the shared_ptr — Pitfall 2: re-resolve at firing time, validate capability AND connectedness when the timer fires).
  - Inside the lambda: same open + cast + null-check sequence as setSystemTimeOn, but on failure/null emit no toast (auto-sync UX = glyph only per D-02).

  Update the upstream test mock pattern accordingly: mock returns `std::shared_ptr<IDevice>` from a fake `DeviceRegistry::open()` (or pass the mock through a registry shim — whatever the upstream test uses, ensure it's adapted to shared_ptr).

**No change to the QSettings load-time validation, the signal shape, or the QML registration approach.**
