<objective>
Make every Stream Dock backend honestly advertise `Capability::Clock` and return `TimeSyncResult::NotImplemented` from `setTime()` — no crashes, no false success, no log spam. Each backend gets a file-static `std::once_flag` that gates a WARN ("setTime() not yet implemented for <codename>") so the first invocation produces one informative log and every subsequent call returns `NotImplemented` silently (Pitfall 14).

This plan is a thin pointer-plan: it adopts **Task 2 of the upstream plan at `docs/superpowers/plans/2026-05-13-time-sync.md` (lines 186-365)** — with two amendments that account for changes since the upstream plan was written (commit `62da68c`):

- **A-02 (D-01 amendment 1)**: AKP815 backend now exists; apply the AKP153 pattern to `akp815.cpp` as a fourth backend.
- **A-03 (D-01 amendment 2 + D-03)**: `register.cpp` now has 17 rows (the canonical AKP153 + AKP153R + AKP153E V2 + AKP03 canonical + 7 AKP03 rebadges + AKP05 + Mirabox N4 + AKP815 + 2 legacy placeholders) — set `.hasClock = true` on **every** row, not just the 5 the upstream plan listed.

D-04 from Phase 4 (rebadges share factory) guarantees that setting `.hasClock = true` on a rebadge row is sufficient — the rebadge construction path uses `Akp03Device` (or the relevant canonical) which already inherits `IClockCapable` per this plan. No per-rebadge code changes needed.
</objective>
