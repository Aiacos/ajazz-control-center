```
SECTION 1 — "shared_ptr survives Removed (Pitfall 1 / SC1)":
- Build a DeviceRegistry with no MockHidEnumerator (we test the cache behaviour directly).

- Call a method on `dev` (e.g. `dev->poll()`); assert it returns without crashing — this is the mechanical proof that the shared_ptr keeps the backend alive past the Removed event.

SECTION 2 — "weak_ptr cache returns the same instance (D-06)":
- Construct a registry, register a test backend factory.
- `auto a = registry.open(id); auto b = registry.open(id);` — assert `a.get() == b.get()` (same instance, flyweight working).
- `a.reset(); b.reset();` then `auto c = registry.open(id);` — assert `c.get() != original_pointer_value` is unverifiable cleanly, but assert that calling open() after both shared_ptrs drop produces a fresh shared_ptr with use_count() == 1 (the cache evicted passively).

SECTION 3 — "300ms debounce coalesces 4 same-key events into 1 (HOTPLUG-05 / D-05)":
- Create a HotplugDebouncer (parent = QObject scope).
- Connect a counter to its `coalesced` signal.
- Call `debouncer.observe(ev)` 4 times in quick succession with the same (vid, pid, serial).
- `QTest::qWait(400);` (or QSignalSpy::wait(400)).
- Assert the counter == 1 — exactly one coalesced emission.

SECTION 4 — "Device shuffle: 3 distinct keys produce 3 emissions, not 9 (per-key isolation)":
- In a loop, fire 3 Removed + 3 Arrived events for 3 different (vid, pid) keys, alternating, all within 100ms.

- Assert exactly 3 coalesced emissions (one per key — the latest event per key).

SECTION 5 — "DeviceModel emits dataChanged({ConnectedRole}) for affected row only (D-03 / SC2)":

- Use `QSignalSpy` on `DeviceModel::dataChanged`.
```
