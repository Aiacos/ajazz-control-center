## 4. Tech Debt Aggregated

### Phase 1: SEC-003 Plugin Host

**Security-deferred (Win32-only):**

- **CR-01** — `out_of_process_plugin_host_win32.cpp:463-467` calls `_putenv_s` for PYTHONPATH/PYTHONDONTWRITEBYTECODE/PYTHONUNBUFFERED in the parent process before `CreateProcessW(lpEnvironment=nullptr)`. Pollutes parent env across host instances and into the sibling manifest verifier subprocess. **Fix path:** per-spawn UTF-16 env block + `CREATE_UNICODE_ENVIRONMENT`. **Size:** ~40-60 LoC. **Why deferred:** untestable from Linux dev environment; needs Windows CI to exercise. Documented in `.planning/phases/01-sec-003-plugin-host/01-FIX-DEFERRED.md`.
- **WR-01** — `loadTrustRoots` mini-grep parser fragility. Partial fix landed in `1fbb46b` (window widening + Catch2 case for malformed JSON). Full fix requires architectural decision among: add `nlohmann::json` dep / break COD-031 / write 80-LoC custom scanner.

**Doc drift:**

- **DOC-02** — Phase 1 `SUMMARY.md:53` claims "PUBLIC visibility" on plugins headers; actual mechanism is `target_include_directories(... PRIVATE ...)` on the app binary. Functionally equivalent (binary targets don't propagate include dirs), but the SUMMARY phrasing is technically inaccurate.

### Phase 2: QML Singleton Sweep

**Doc drift:**

- **DOC-01** — Phase 2 `SUMMARY.md:13, 27` describes the pattern as `qmlRegisterSingletonInstance`. The actual code uses `QML_SINGLETON` + private ctor + static `create(QQmlEngine*, QJSEngine*)` returning the `registerInstance()`-handed C++ instance. Both are valid Qt 6 patterns achieving identical semantics; the SUMMARY phrasing misrepresents the mechanism but not the outcome.
