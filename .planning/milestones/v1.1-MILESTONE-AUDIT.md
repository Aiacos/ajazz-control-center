## Parallel Execution Lessons

Three concurrent execute agents (Phase 4 + Phase 6 + Phase 7) created two distinct git-coordination issues:

1. **Phase 7's atomic-commit split** — Phase 4 + Phase 6 agents ran `git reset` operations during Plan 07-01 execution, wiping Phase 7's unstaged changes. The `manifest_signer_common.{hpp,cpp}` files were re-applied and ended up in Phase 4 agent's `e482152` commit rather than Phase 7's `e094239`. Functionally preserved (both commits land on `main`; reverting both undoes the parser swap cleanly) but the audit trail is split.
1. **Phase 5 planner `--no-verify`** — pre-commit's stash/restore mechanism failed when a planner agent wanted to commit while 3 execute agents had unstaged work. The planner used `--no-verify` as a workaround (after independently verifying the 8 plan files pass all hooks). Violates the no-skip-hooks principle but the bypass was tooling-mechanical, not content-substantive.

**Recommendation for next milestone:** cap concurrent execute agents at 2.
