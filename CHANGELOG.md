### Added

- Vendor reverse-engineering & parity research scaffolding under `docs/research/`: four documents wire up the multi-day track filed in `e846ddb`. `vendor-software-inventory.md` catalogues 30+ official AJAZZ desktop apps with URL / size / Last-Modified / Content-MD5 (Stream Dock AJAZZ Windows + macOS, AK680/820/870 + AKS/AKP/NK keyboards, AJ199/159/179/139/039/129/358/52 + AM3 mice) — all metadata captured via HEAD probes against the vendor CDNs (Aliyun OSS + shopify + epomaker mirrors), no binaries pulled into the repo. `vendor-protocol-notes.md` ships the clean-room methodology (capture metadata table, tool checklist with explicit install + remove steps so the recon host stays clean, protocol-description shape) plus per-device-family section skeletons that fill in as USB / WebSocket / IPC captures land. `vendor-feature-matrix.md` is the gap analysis with rows seeded from `docs/_data/devices.yaml` and public feature sheets across Stream Dock keys / encoders / screens, keyboard RGB / macros / layers, mouse DPI / RGB / buttons, and cross-cutting infrastructure (single-instance, tray, autostart, themes, plugin store) — ✅ / 🟡 / ❌ / ❓ cells goal-backwards (❓ flips only when a `capture-id` confirms vendor behaviour). `vendor-techniques.md` lists 6 candidate cross-pollination techniques ranked by expected stability impact (HID reconnect debounce, firmware update retry / rollback, per-device USB chunk sizing, profile sync conflict resolution, telemetry beaconing, crash-safe persistence). `docs/research/README.md` is the index + the one place that states the clean-room and no-redistribution rules every contributor to this directory must follow.
- Settings page (`src/app/qml/Settings.qml`): Material-styled controls for theme mode (Auto/Light/Dark), launch-on-login, and start-minimised. Bound to existing `ThemeService` / `AutostartService` / `TrayController` properties.
- AJAZZ wordmark visible in `AppHeader.qml`: shipped at `qrc:/qt/qml/AjazzControlCenter/branding/ajazz-logo.png`, rendered with PreserveAspectFit at 32 px height. Falls back to the product-name text label when the image fails to load (e.g. branded builds without a logo asset).
- Brand-aligned application icon: `resources/branding/app.svg` redesigned (clean-room, by Aiacos) to evoke the AJAZZ visual identity — a triangular "A" monogram with an inner droplet element rendered in the brand red (`#E63946`) on a dark rounded square. Replaces the previous 3x3 macropad-grid placeholder. Tray icon (`resources/branding/tray.svg`) updated to a matching stroked silhouette so the brand stays recognisable down to 16 px on light and dark taskbars. All raster derivatives (Windows `.ico`, macOS `.icns`, Linux hicolor PNG fallbacks, tray PNG fallbacks) regenerated from the new SVGs. README and wiki landing page now show the brand mark.
- Plugin SDK foundation: `docs/schemas/plugin_manifest.schema.json` (JSON Schema Draft 2020-12) is the authoritative manifest contract — a strict superset of the Stream Deck SDK-2 manifest, with `CodePathLin` / `CodePaths` for OpenDeck compatibility and an `Ajazz` namespace for sandbox / permissions / Sigstore signing / device-class scoping. Companion architecture document at `docs/architecture/PLUGIN-SDK.md` covers deployment shape, discovery order, lifecycle (negotiation → spawn → register → steady-state → shutdown), wire protocol (line-delimited JSON over WebSocket), per-OS sandboxing (bwrap + seccomp on Linux, `sandbox-exec` on macOS, AppContainer on Windows, `flatpak-spawn --sandbox` inside Flatpak), permission model, Sigstore supply-chain story, signed-catalogue Plugin Store and the `acc plugin scaffold/run/pack/lint` developer CLI.
- Plugin Store UI (`src/app/qml/PluginStore.qml`): Material-styled browse page mounted in a right-edge modal `Drawer` opened from the new “Plugins” button in `AppHeader`. Virtualised `GridView` of plugin tiles with name / version / author / one-line summary / colour-coded compatibility badge, install + uninstall + enable-toggle controls and a side-sheet details pane reading `pluginCatalog.entryFor()` with a revision counter that re-resolves the QVariantMap on every mutation. Source switcher tabs **All / Installed / AJAZZ Streamdock / Community** scope the catalogue, with an info banner on the Streamdock tab explaining that those plugins are mirrored from the upstream catalogue and run through the same sandboxed lifecycle as native plugins. Backed by `PluginCatalogModel` (`src/app/src/plugin_catalog_model.{hpp,cpp}`), an 11-row mock fixture exercising every compatibility mode (`native`, `opendeck`, `streamdeck`, `streamdock`) and source (`local`, `community`, `streamdock`).
- AJAZZ Streamdock store integration (compatibility mode `streamdock`): manifest schema gains `Ajazz.Compatibility.Mode = "streamdock"` plus an opaque `StreamdockProductId` resolved by the catalogue mirror to the upstream signed bundle URL, and `docs/architecture/PLUGIN-SDK.md` documents the dialect (Stream Deck SDK-2 superset with the `streamdock/*` event namespace for hardware dial widgets and per-key haptics).
- `feat(plugins)`: live Streamdock catalogue mirror with offline fallback. New `StreamdockCatalogFetcher` (`src/app/src/streamdock_catalog_fetcher.{hpp,cpp}`) walks the upstream Space catalogue (`POST https://space.key123.vip/interface/user/productInfo/list`) page by page, accumulates the plugin records, translates upstream `deviceUuid`s (`293`, `293E`, `N3`, `N4`, `D92`…) to AKP codenames (`akp153`, `akp153e`, `akp03`, `akp815`, `ak980pro`) via a curated table, and writes an atomic `QSaveFile` mirror to `<XDG_CACHE_HOME>/ajazz-control-center/streamdock-catalog.json`. Three resolution layers — live → on-disk cache → bundled `resources/streamdock-fallback.json` (compiled into the binary) — guarantee the AJAZZ Streamdock tab is never empty (first launch on an air-gapped host or CI smoke runs still get rows). `PluginCatalogModel` exposes `streamdockState` (`loading` / `online` / `cached` / `offline`) and `streamdockFetchedAtUnixMs` to QML, and `PluginStore.qml` renders a live status pill on the tab banner with a rolling relative-time label ("updated 3 min ago"). Endpoint is overridable via the `ACC_STREAMDOCK_CATALOG_URL` env var (set to `disabled` to skip the network fetch entirely on air-gapped builds). Pure helpers (`parseUpstreamJson`, `mapUpstreamDevices`, `humaniseSize`, `deriveUuid`) plus the cache round-trip and bundled fallback are covered by `tests/unit/test_streamdock_catalog_fetcher.cpp`.
- Property Inspector M1 + M2 + M3 + M4 — plugin-authored HTML Property Inspector panel embedded in the inspector pane. M1 introduced `src/app/src/property_inspector_controller.{hpp,cpp}` exposed to QML as `propertyInspectorController` (stub `loadInspector` / `closeInspector` / `webEngineAvailable` / `hasHtmlInspector` API surface plus the `AJAZZ_BUILD_PROPERTY_INSPECTOR` CMake gate). M2 wired the actual Qt WebEngine surface: `loadInspector` lazily creates a `QWebEngineProfile` per plugin UUID (so two plugins never share cookies / storage), spins up a fresh `QWebEnginePage` with a conservative `QWebEngineSettings` baseline (no remote URLs from local content, no clipboard, no fullscreen, no plugins, no screen capture), loads the HTML via `file://`, and exposes the page to QML via the new `activePage` Q_PROPERTY (forward-declared with `Q_DECLARE_OPAQUE_POINTER` so the header still compiles on minimal Qt installs without WebEngineQuick). `PropertyInspector.qml` becomes a `Loader`-based switcher between the new `PIWebView.qml` (only compiled when `AJAZZ_HAVE_WEBENGINE` is set) and the schema-driven `NativePropertyInspector.qml` (extracted verbatim from the original `PropertyInspector.qml` body so it remains the universal fallback). `main.cpp` calls `QtWebEngineQuick::initialize()` before `QApplication` when the macro is set. M3 adds the QWebChannel `$SD` JS bridge: a new `PIBridge` QObject (`src/app/src/pi_bridge.{hpp,cpp}`) ships the full Stream Deck SDK-2 PI method surface (`setSettings`, `getSettings`, `setGlobalSettings`, `getGlobalSettings`, `sendToPlugin`, `setTitle`, `setImage`, `openUrl`, `logMessage`) and signals (`didReceiveSettings`, `didReceiveGlobalSettings`, `sendToPropertyInspector`, `connected`); the controller registers it on a per-page `QWebChannel` under the name `"$SD"` before each `page->load(...)` so a PI HTML page that includes `qrc:///qtwebchannel/qwebchannel.js` and instantiates `new QWebChannel(qt.webChannelTransport, ...)` gets the same `$SD` global an Elgato Stream Deck or OpenDeck PI expects. M3 implements the bridge as a logging stub on the host side. M4 wires `setSettings` / `getSettings` / `setGlobalSettings` / `getGlobalSettings` through to atomic on-disk JSON: per-context settings land at `QStandardPaths::AppDataLocation/plugins/<pluginUuid>/settings/<contextUuid>.json` and plugin-wide settings at `QStandardPaths::AppDataLocation/plugins/<pluginUuid>/global.json` (on Linux: `~/.local/share/AjazzControlCenter/plugins/...`). Writes go through `QSaveFile` so a crash mid-write cannot corrupt the destination (write-to-temp + atomic rename, same pattern `src/core/src/profile_io.cpp` uses for profile JSON). Untrusted JS-supplied uuid components are validated against a path-traversal whitelist before being concatenated into a filesystem path: empty values, `.`, `..`, embedded slashes/backslashes, NUL or ASCII control bytes are rejected with a logged error. Per-context payloads are capped at 1 MiB and parsed via `QJsonDocument::fromJson` before the write, so a runaway plugin cannot fill the disk or stash invalid JSON. The `$SD` API surface is unchanged: getter methods stay async, returning via the existing `didReceiveSettings` / `didReceiveGlobalSettings` signals so existing PI HTML pages keep working unchanged. M5 will route `sendToPlugin` / `sendToPropertyInspector` over the plugin host's WebSocket connection.
- README + Wiki Material UI screenshots (`docs/screenshots/main-{dark,light}.png`): captured against the running app with `Appearance/Mode` preset in `QSettings`, full HiDPI 3792 × 2061 each. README hero uses a `<picture>` element with `prefers-color-scheme` so the image adapts to the visitor's GitHub theme; `docs/wiki/Home.md` shows both modes side by side under a new `## Screenshots` section. Surfaced an unrelated light-theme contrast bug (device-card delegate ignoring `themeService.mode`) — tracked under TODO → UI polish.
- App-icon helper `ajazz::app::makeAppIcon` (`src/app/src/app_icon.{hpp,cpp}`): builds a multi-resolution `QIcon` by rasterising the canonical `branding/app.svg` natively at every Qt-consumed size (16/22/24/32/48/64/128/256/512 via `QSvgRenderer`) instead of letting `QIcon` downscale a single high-res raster. Used by both the tray and the window/decoration icon path so each surface gets a sharp pixmap drawn straight from the vector; also serves as the `IconPixmap` SNI fallback for legacy hosts that do not honour `IconName` lookups.

### CI / automation

- `pre-commit` gains a `check-jsonschema` stage that validates every committed JSON Schema against its Draft 2020-12 metaschema and every plugin `manifest.json` against `docs/schemas/plugin_manifest.schema.json`. Manifest drift is now caught at commit time, not in CI.
- `pre-commit` clang-tidy hook now also runs at the `pre-push` stage: `make precommit` installs the matching git hook so static analysis fires once per push (catching warnings that the cheap per-commit hooks can't see), instead of waiting for CI. The hook stays cheap-on-commit because clang-tidy is not in the per-commit stage list. New `make tidy-fix` Makefile target applies clang-tidy `--fix --fix-errors` to staged C++ files for contributors who would rather have the modernize-* / readability-* checks rewrite their code than hand-fix one warning at a time. CONTRIBUTING.md gains a "Linters and the auto-fix flow" section walking through every git hook stage and what runs where.
- New weekly workflow `precommit-autoupdate.yml`: runs `pre-commit autoupdate --freeze`, re-runs every hook so the resulting auto-fixes ride the same PR, and opens a labelled pull request via `peter-evans/create-pull-request`. Combined with the existing Lint workflow's auto-fix push, the human reviewer sees only a green PR.
- `.typos.toml` now ignores image-format plural acronyms via the regex `^[A-Z]{2,5}s$` (typos was rewriting the literal sequence P‑N‑G‑s into the wrong word) and adds Plugin SDK vocabulary (Sigstore, Fulcio, Rekor, bwrap, OpenDeck, seccomp, AppContainer, QtWebEngine, WebView2, WKWebView).
- `mdformat` exclude list now covers `docs/architecture/*.md` so wide tables in the architecture docs are not reflowed into ping-pong diffs.
- Per-platform desktop / file-explorer icons. Generated from `resources/branding/app.svg`: `resources/icons/app.ico` (multi-size 16/24/32/48/64/128/256 for Windows .exe + WiX installer), `resources/icons/app.icns` (10 entries spanning 16-512 px for macOS bundle), and `resources/icons/hicolor/app-{16,24,32,48,64,128,256,512}.png` (Linux hicolor theme PNG fallbacks). Linux desktop entry installs each PNG to `share/icons/hicolor/<size>x<size>/apps/` so GNOME / KDE / GTK launchers and AppStream metadata viewers all find a crisp raster without depending on the SVG renderer.
- Tray icon refreshed: monochrome `currentColor`-based `resources/branding/tray.svg` plus white-on-transparent PNG raster fallbacks at 16/22/24/32/44/48 px under `resources/icons/tray/`. Renders correctly on light and dark menu-bars / system trays.
- Windows `.exe` now ships the application icon via an auto-generated `.rc` file (`IDI_ICON1 ICON DISCARDABLE app.ico`) so Explorer, the taskbar, and the Start Menu shortcut display the brand icon. macOS bundle is wired through `MACOSX_BUNDLE_ICON_FILE`. Runtime `QApplication::setWindowIcon()` covers Linux/X11/Wayland and serves as a fallback elsewhere.

### Changed / Hardened

- Plugin host (`src/plugins/src/plugin_host.cpp`): rejects plugin directories whose basename collides with a Python stdlib module name (SEC-S2 hardening, uses `sys.stdlib_module_names`); deduplicates `sys.path` entries across repeated `loadAll()` calls (SEC-S3).
- Profile IO (`src/core/src/profile_io.cpp`): Linux/macOS write path switched from `std::ofstream` to `::open(O_WRONLY|O_CREAT|O_EXCL|O_NOFOLLOW|O_CLOEXEC, 0600)` with `fsync()` on the original fd and `fsync()` of the parent directory after the atomic rename (SEC-S4 + SEC-S6 hardening). Eliminates symlink-redirect races and a buffered-stdio durability gap.
- Profile IO atomic rename on Windows: now uses `ReplaceFileW` for the existing-destination case (purpose-built for atomic-replace-with-backup, dodges most AV/indexer races) and falls back to `MoveFileExW` only when the target doesn't exist yet. Retry budget widened to also catch `ERROR_LOCK_VIOLATION` and `ERROR_USER_MAPPED_FILE`.
- DeviceRegistry switched from a Meyers-singleton to constructor-injected ownership (audit finding A1). `ajazz::app::Application` now holds a `core::DeviceRegistry m_deviceRegistry` data member and threads it through every `registerAll(core::DeviceRegistry&)` backend bootstrap (`streamdeck`, `keyboard`, `mouse`) plus the new `DeviceModel(registry, parent)` constructor — so per-test isolation, no hidden global state, and no startup-order foot-gun where a backend's `instance()` call fires before the Application is even constructed. The previous `DeviceRegistry::instance()` symbol is kept as a `[[deprecated]]` transition shim returning a process-wide fallback so any not-yet-migrated translation unit keeps compiling; the deprecation message embeds the audit-finding label to make grep-based migration cleanup trivial. `tests/unit/test_device_registry.cpp` was rewritten to construct its own local `DeviceRegistry` and gains a new `[registry][isolation]` case proving two locally-owned instances don't share state. All 75 existing tests still green; full build + ctest suite verified on the `dev` preset.
- Logger gains a pluggable `LogSink` abstraction (audit finding A5). `ajazz::core::LogSink` is the new abstract base; the default `StderrSink` reproduces the legacy `[<ms>] [<LEVEL>] [<module>] <message>` formatting. `setLogSink(shared_ptr<LogSink>)` swaps the active sink atomically against concurrent `log()` calls — internally an `std::atomic<std::shared_ptr<LogSink>>` so the previous sink stays alive past the swap if a `log()` call is in flight. The `AJAZZ_LOG_TRACE/DEBUG/INFO/WARN/ERROR` macros are unchanged so every existing call site keeps working unmodified; tests can now install a capturing sink and assert on what subsystems logged (`tests/unit/test_logger.cpp` ships a reusable `CapturingSink` + `ScopedSink` RAII helper plus two new cases that pin the level filter / sink dispatch contract), and a future deployment can route to journald / syslog / a Qt model without touching translation units.
- ActionEngine threading model (audit finding A2). `ActionEngine::run()` no longer calls `std::this_thread::sleep_for` on the caller — `Sleep` steps and non-zero post-step `delayMs` waits are now deferred via a pluggable `ajazz::core::Executor` so the HID poll thread / Qt main thread are freed immediately while the chain continuation runs on a worker. Core stays Qt-free: the abstraction is plain C++, with a `BlockingExecutor` default that preserves legacy in-place-sleep semantics for tests and headless callers. The GUI app ships `ajazz::app::QtExecutor` (backed by `QTimer::singleShot` with a QObject context, so destruction cancels in-flight tasks) for the production wiring path. Three new unit tests cover the contract: `BlockingExecutor` runs continuations inline, a fake async executor demonstrates that `run()` returns to its caller before the chain finishes, and post-step `delayMs` is also routed through the executor.
- EventBus refactored to copy-on-write of an immutable subscriber vector (audit finding A3). Subscribers are now held in a `std::atomic<std::shared_ptr<vector<Subscription, Handler> const>>`; `publish()` is lock-free (atomic-load + iterate the immutable snapshot), the cost shifts to the rare subscribe / unsubscribe paths which serialise on a write-only mutex and rebuild the vector once. Eliminates the per-`publish()` copy of the previous `unordered_map<Subscription, Handler>` snapshot — material on encoder-heavy devices like the AKP05 that emit >100 events/s. Existing 4/4 EventBus tests still pass; the public `subscribe` / `unsubscribe` / `publish` API surface is unchanged so all call sites stay untouched.
- System tray icon visibility on Wayland — fixed the silent invisibility under niri / Quickshell, KDE Plasma, Sway and other StatusNotifierItem-based bars. Two-part change: (1) `setDesktopFileName` now uses the simple `"ajazz-control-center"` slug that matches the file installed by the Linux CMake rule, instead of the reverse-DNS app id, so xdg-desktop-portal and Wayland tray hosts can resolve our app metadata (no more `App info not found` errors); (2) the tray + window `QIcon` is constructed via `QIcon::fromTheme("ajazz-control-center", makeAppIcon(...))` so the SNI item exposes a populated `IconName` property — most Wayland tray hosts honour only `IconName` and silently drop pixmap-only icons. The multi-size pixmap rasterised via `makeAppIcon` is kept as the embedded fallback for hosts that do honour `IconPixmap` (X11 trays, GNOME Shell + AppIndicator). Verified at runtime via `busctl --user`: SNI item now exposes `IconName='ajazz-control-center'` plus 6 ARGB pixmap entries.
